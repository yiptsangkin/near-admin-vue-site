# 布局

near-admin采用了比较传统的中后台布局方式，与此同时也将各个模块进行了解耦区分，将其纳入到core中由官方维护，一般情况下不建议对其进行修改，这样以后更新会比较麻烦。

# Layout

# 以下为官方的布局组件
├── NAside.vue                      # 侧边栏
├── NBreadCrumb.vue                 # 面包屑
├── NHeader.vue                     # 头部
├── NKeepAlive.vue                  # 缓存组件
├── NManage.vue                     # 管理内容组件
├── NNoFound.vue                    # 404
├── NNoRight.vue                    # 无权限
├── NTag.vue                        # 标签栏
├── NTagContentMenu.vue             # 右键菜单
├── NWebView.vue                    # webview组件

布局如下图所示，布局的组件都在core文件夹中，由于在alias中已经配置了core中组件的映射，因此在但文件组件中使用@corecp就可以映射对应的组件路径。

// @corecp映射了core文件夹中的组件
import NBreadCrumb from '@corecp/NBreadCrumb.vue'

由于near-admin是一个致力于尽量不进行路由配置的中后端框架，所以，在标签页的右键菜单方面还提供了独立页面的功能，能够在浏览器标签中动态生成一个新的独立页面。

# NAside组件

NAside@corecp/NAside.vue主要提供侧边菜单的功能，默认只支持最大三层菜单结构

# NHeader组件

NHeader@corecp/NHeader.vue组件中包含了系统标题、菜单分组、全屏操作、国际化以及用户信息，用户信息中又包括了浏览器端的前端日志、个性化设置、清空配置以及推出登录功能。

# NBreadCrumb组件

NBreadCrumb@corecp/NBreadCrumb.vue提供了面包屑功能，系统方面默认只会为菜单中打开的页面设置面包屑，如果通过api创建页面也想要自定义一些面包屑的话也可以直接在创建页面的时候在参数中进行配置。

# NManage组件

NManage@corecp/NManage.vue是near-admin的核心组件，它是通过动态异步组件来实现标签页的功能，更加的灵活方便。

# NTag、NTagContentMenu组件

NTag、NTagContentMenu@corecp/NTag.vue @corecp/NTagContentMenu.vue组件主要提供的是标签栏以及标签右键功能，框架内部已经提供了相关的右键操作，能够满足觉得部分的场景，后期可能会增加部分更加有趣的功能项。

# NKeepAlive组件

NKeepAlive@corecp/NKeepAlive.vue组件是基于Vue官方KeepAlive组件进行改造的组件，主要是解决component组件动态渲染异步组件以及同名组件的缓存问题，支持根据key缓存页面组件内容。

# NNoFound、NNoRight组件

NNoFound、NNoRight@corecp/NNoFound.vue @corecp/NNoRight.vue组件是用于404页面或者访问页面不再权限组中的页面进行提示的组件。
