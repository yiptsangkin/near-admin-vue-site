# 权限

# 页面权限

near-admin是基于动态组件实现的标签页，因此，在权限控制上主要对异步组件的权限控制，near-admin在全局状态中添加了rightPathList用于维护拥有访问权限的组件路径， 框架内部会将侧边栏菜单的组件地址进行去重后存入到rightPathList，如果用户需要自行控制权限地址的话可以将用户拥有权限的地址加入到rightPathList中。具体可以使用 vuex中的actionchangeRightPathList进行rightPathList的修改。

提示

通过全局指令$newpage进行新增的标签页默认是拥有权限的。如果想要对这段逻辑进行控制的话，可以在@corets/plugin.ts进行需改。

// set apiNew to true
if (cpInfo.params) {
    cpInfo.params.apiNew = true
} else {
    cpInfo.params = {
        apiNew: true
    }
}

如果用户没有该组件的访问权限，框架自身会返回403界面。

403界面

如果用户访问的地址是不存在的页面，则会出现404页面。

404界面

一般来说前端控制权限其实意义不大，所以near-admin基本上是基于后端返回的菜单内容进行权限的控制。

# 指令权限

near-admin提供了权限指令，方便开发者能够根据用户的角色对一些dom进行的权限操作。同时还支持逻辑非的功能，更加灵活的配置权限。

<!-- 用户组在admin（管理员）或者salesman（业务员）中的显示这个dom -->
<div v-permit="['admin', 'salesman']">管理员、业务员</div>

<!-- 用户组不在admin（管理员）和salesman（业务员）中的显示这个dom -->
<div v-permit="['!admin', '!salesman']">非管理员，非业务员</div>

<!-- 用户组不在admin（管理员）,在salesman（业务员）或tester（测试员）中的显示这个dom -->
<div v-permit="['!admin', 'salesman', 'tester']">非管理员，业务员或测试员</div>
