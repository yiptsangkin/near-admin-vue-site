# 新增页面

near-admin本身是一个支持多页打包的应用，意味你可以在一个工程中创建多个单页应用，在cli开发完成之前，near-admin提供了简单的创建单页应用的脚本，只 需要在项目目录下面运行npm指令，输入对应的单页应用的名称即可：

npm run create:page
? Create a page application which named: › Your application name

# 新增管理标签页组件

near-admin对于标签组件有一个自己的规范，它可以方便你使用框架中一些常用的全局状态。标签组件的基础内容如下：

<template>
</template>

<script lang="ts">
    import Vue from 'vue'
    import CoreBase from '@corets/base'
    import Base from '@custom/base'

    export default Vue.extend({
        name: 'ComponentName',
        mixins: [CoreBase, Base],
        data () {
            return {
            }
        }
    })
</script>

<style lang="scss" scoped>
</style>

实际开发过程中，可以通过在webstorm和vscode设置一些代码模板，这样子开发起来相对比较方便简单。

# 通过框架内置的api创建一个标签

对于near-admin来说，一个标签页其实就代表着一个页面组件，框架内部提供两个简单的指令用于控制页面，一个是$newpage，一个是$closepage，这两个指令 基本上可以满足开发者对于标签页面的一些操作控制。

const self = this
self.$newpage({
    title: 'your tag title',
    component: 'your tag component path'
})

具体api允许支持的其他参数内容，可以通过框架全局的类型文件进行查看。

# 新增页面api

对于near-admin来说，页面使用到的接口我们建议在assets里面的api中维护，而且最好按照功能类型进行分类，同时每一个模块的api需要分两个文件维护，分别 是apiUrl.ts和apiMethod.ts两个，一个用于维护请求的地址，一个是用于维护请求的方法。

// apiUrl.ts
import comConfig from '@custom/config'
const apiHost = comConfig.apiHost
export default {
    getSendMsg: `${apiHost}/mock-data/getSendMsg`,
    checkLoginByAccount: `${apiHost}/mock-data/checkLoginByAccount`,
    checkLoginByPhone: `${apiHost}/mock-data/checkLoginByPhone`
}

// apiMethod.ts
import apiUrl from './apiUrl'
import utils from '@corets/utils'
import {ReqType, ResType} from '@corets/type'

const getSendMsg = async (reqType: ReqType) => {
    reqType.method = 'POST'
    reqType.url = apiUrl.getSendMsg
    const result = await utils.sendReq(reqType)
    return result
}

const checkLoginByAccount = async (reqType: ReqType) => {
    reqType.method = 'POST'
    reqType.url = apiUrl.checkLoginByAccount
    const result = await utils.sendReq(reqType)
    return result
}

const checkLoginByPhone = async (reqType: ReqType) => {
    reqType.method = 'POST'
    reqType.url = apiUrl.checkLoginByPhone
    const result = await utils.sendReq(reqType)
    return result
}

export default {
    getSendMsg,
    checkLoginByAccount,
    checkLoginByPhone
}

提示

在网络请求库方面，near-admin基于axios上重新封装了请求，支持async/await和回调两种模式，使用回调模式的话，会默认帮你处理好success，fail，error 事件的判断，而如果使用async/await的模式，就需要自行判断code。near-admin的请求库默认successcode: 0，failcode: else number， errorjs error or http request error, such as 500, 404

// async/await
const result = await api.checkLoginByAccount({
    data: {
        username,
        password
    }
})
console.log(result)
// callback
api.checkLoginByAccount({
    data: {
        username,
        password
    },
    success: (res: any) => {
        self.$message.success(
            self.$t(dict.localeObj.loginForm.loginSuccess) as string,
            2
        ).then(() => {
            self.toManage()
        })
    },
    fail: (res: any) => {
        self.$message.error(res.message)
    }
})

# 新增样式

对于near-admin来说，作者个人不喜欢把样式写在sfc中，因为这样子有些组件内部的样式想要引用到其他页面就比较麻烦，因此个人推荐是在asset中的sass进行 维护，当然这里也主要是看大家的使用习惯，没有非常强制的要求。框架内部本身内置了一些mixin和params，开发者可以按照需求选用或者完善。详细使用方法 请参照样式章节
