# 样式

# 开发风格约定

near-admin中不太推荐在sfc中维护样式内容，就跟页面组件中near-admin推荐将一些公用的方法通过base、core进行mixin一样，因为第一，写一些样式比较复杂的页面时， 整个sfc文件会非常的臃肿，第二，这样子写如果样式要复用的时候没那么方便。所以一般来说，作者推荐通过单文件来维护样式。在near-admin的文件架构中，开发者可以将 scss样式文件写在assets/scss中，并按照功能页或者组件对样式目录进行划分。

提示

在sfc中引用样式文件的时候可以通过映射进行引用，框架本身将@corescss指向core中的scss文件夹，将@scss指向asset中的scss文件夹，开发者在使用过程中可以合理的使用 映射，避免写出过于冗长的相对路径。

@import "~@corescss/params.scss"

框架本身依据hsl计算了各种颜色深浅明暗的参数，对于一些颜色方面没有要求跟设计稿一致的情况下，可以直接使用框架内部的变量。

@mixin ellipse ($line) {
  overflow: hidden;
  text-overflow: ellipsis;
  -webkit-box-orient: vertical;
  -webkit-line-clamp: $line;
}

框架本身的mixin提供了多行缩略的方法。

# 主题色维护

主题色维护方面，除了在vue.config.js中配置的antd的主题色，还需要调整assets/scss/theme中的主题色（$primary-color）。

// vue.config.js
css: {
    extract: {
        ignoreOrder: true
    },
    loaderOptions: {
        less: {
            lessOptions: {
                javascriptEnabled: true,
                modifyVars: {
                    'primary-color': '#41B783', // primary-color
                    'link-color': '#41B783', // link-color
                    'border-radius-base': '2px', // border-radius-base
                }
            }
        }
    }
}
// theme.scss
@import "~@corescss/params.scss";
$primary-color: $vue-green;

# 覆盖ant-design

通常来说，一般我们都不建议全局的修改ant-design的样式，所以个人比较推崇与在sfc中使用::v-deep对相关的需要重写的ant-design样式进行覆盖。

::v-deep .ant-transfer{
    display: flex;
    align-items: center;
}
